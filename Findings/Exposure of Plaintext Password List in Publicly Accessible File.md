# Exposure of Plaintext Password List in Publicly Accessible File
Severity: High

CVSS 3.1: CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:N/A:N — 7.5

CWE: CWE-538 (Insertion of Sensitive Information into Externally-Accessible File or Directory)

(ergänzend: CWE-312 — Cleartext Storage of Sensitive Information; CWE-261 — Weak Encoding/Cleartext for Passwords)
Description:

A publicly accessible file (/decoda9013smith21985.txt), explicitly referenced in robots.txt, contains 651 plaintext password values generated by a "Password Generator Prototype". The file is reachable without authentication. Referencing it in robots.txt advertises rather than protects the path. The values are stored in cleartext (not hashed or encoded).
Affected Component:

http://10.10.10.10/decoda9013smith21985.txt (referenced in /robots.txt)
Proof of Concept:

Retrieve http://10.10.10.10/robots.txt, which lists the path. (Screenshot)
Access http://10.10.10.10/decoda9013smith21985.txt directly without authentication. (Screenshot)
The file returns 651 cleartext password strings. (Screenshot — values redacted in report)

Impact:

An unauthenticated attacker obtains a list of 651 cleartext passwords. If any of these values are assigned to active accounts, they enable direct account compromise via credential testing. Even absent a confirmed account match, exposing generated credential material in cleartext through a publicly reachable file represents a severe confidentiality failure and a likely precursor to account takeover. The values appear to be generated passwords; mapping them to specific active accounts was not performed, as exhaustive credential testing was deemed out of scope for a controlled assessment.
Remediation:

Remove the file from the web root immediately. Never store credential material in cleartext, and never reference sensitive paths in robots.txt. If any listed values are or were ever active passwords, rotate all affected credentials. Store secrets outside the web-accessible directory with encryption and strict access control. Enforce password hashing (e.g. bcrypt/Argon2) for any stored authentication data.
